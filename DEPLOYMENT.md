# 部署与商业化上线清单

## 一键部署到 Render

1. 将仓库推送到 GitHub。
2. 在 Render 选择 **New > Blueprint**，连接仓库并确认 `render.yaml`。
3. 为后端服务填写 `API_KEY`，其余变量已在蓝图中配置。
4. 等待前后端服务构建完成，访问 `https://ai-interview-trainer.onrender.com`。
5. 如果 Render 自动生成的服务域名与蓝图名称不同，同步修改：
   - 前端 `VITE_API_BASE_URL`
   - 后端 `FRONTEND_URL`

## 上线前必须补齐

- 当前后端会话保存在内存中，服务重启后会丢失。正式收费前接入 PostgreSQL 或 Redis。
- 当前没有账号、用量限制和支付。建议先接入手机号或邮箱登录，再按训练次数扣减额度。
- 套餐页当前只用于验证购买意愿，不会实际扣费。支付可接入微信支付、支付宝或 Lemon Squeezy。
- 为 LLM 调用增加单用户限流、每日预算上限和失败重试，避免 API Key 被滥用。
- 补充隐私政策、用户协议、数据删除入口，并明确面试录入内容的保存周期。
- 接入基础分析事件：开始训练、JD 解析成功、完成面试、查看报告、点击升级。

## 建议的最小盈利路径

先保留 1 次免费完整体验，用 39 元月卡验证付费。只有当“完成报告后点击升级”的转化稳定后，再开发季卡、历史成长对比与企业版，避免过早建设复杂支付和会员体系。
