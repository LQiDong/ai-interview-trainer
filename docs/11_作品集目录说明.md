## 面试中推荐的展示路径

### 路径 A：产品设计向（适合 AI PM / 产品岗面试）

```
1. 打开 08_面试3分钟讲稿.md → 3 分钟概述
2. 打开 01_PRD_AI面试训练Agent.md → 展示 PRD 中的 Agent Team 设计（第 4 节）
3. 打开 04_Agent工作流设计.md → 展示 Agent 间契约设计
4. 打开 evals/eval_readme.md → 展示评测方法论
5. 打开 05_Badcase_A_评估笼统.md → 展示"发现问题→修复→验证"完整链路
```

### 路径 B：技术架构向（适合有技术面的 AI PM 岗）

```
1. 打开 backend/main.py → 展示 API 结构
2. 打开 prompts/evaluator_agent.md → 展示 Prompt 设计细节
3. 打开 backend/services/evaluator.py → 展示 Agent 实现
4. 打开 evals/run_eval.py → 展示自动化评测
5. 打开 06_Badcase_B_建议脱节.md → 展示跨 Agent 契约问题的诊断
```

### 路径 C：快速扫描（面试官时间紧）

```
1. 3 分钟讲稿
2. 打开 evals/eval_report.md → "这是我们的 AI vs 人工对比结果"
3. 打开一个 Badcase doc → "这是我们发现的一个具体问题及修复"
4. 问面试官想深入哪个部分
```

### 不引用的数字

- ❌ 用户量 / 月活 / 留存率 — 无真实用户数据
- ❌ 面试通过率 — 产品定位明确不承诺通过率
- ❌ 模型效果提升 X% — 未经真实 LLM 环境下的回归评测验证
- ❌ 任何"业界首创""行业领先"等形容词 — 不夸大

## 面试准备检查清单

- [ ] 本地能启动前端 `npm run dev`（`localhost:5173`）— 可随时演示

- [ ] 浏览器书签栏已保存本文件（`docs/11_作品集目录说明.md`），面试官追问时可秒开
