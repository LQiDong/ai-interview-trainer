# 面试评估评分标准 (Rubric)

> **用途**：人工标注和 AI 评估的统一评分参照。所有标注者（含 AI 模型）必须基于此 Rubric 打分，确保评分一致性。

---

## 评分总则

| 规则 | 说明 |
|------|------|
| **评分对象** | 候选人在单道题上的**单次回答**（含同一追问链下的所有子回答） |
| **评分尺度** | 以 `岗位画像` 中的 `position.level` 为参照系——初级岗和专家岗的 3 分标准不同 |
| **证据驱动** | 每条分数必须有对话原文引用作为支撑。不允许"凭印象打分" |
| **独立性** | 四个维度独立评分，相互不耦合。一个维度 0 分不影响另一个维度 3 分 |
| **整数评分** | 分数仅限 0 / 1 / 2 / 3，不允许 1.5、2.5 等小数（维度均分可为小数） |

---

## 维度 1：内容匹配度（Content Relevance）— 权重 35%

> **衡量的是**：回答是否**命中问题的核心考点**和**岗位的核心要求**。

### 评分锚定

| 分数 | 标签 | 标准 | 典型表现 |
|------|------|------|---------|
| **3** | 精准命中 | 全面且精准地命中问题所有考点；展示出**超出岗位预期**的深度理解；能主动关联相关领域知识，形成体系化认知 | "不仅说了 Agent 记忆的类型，还区分了短期/长期/工作记忆的存储策略、检索方式和适用场景，并举例说明了不同场景下的取舍" |
| **2** | 基本覆盖 | 基本命中考点，覆盖了问题的主要方面；展示出与岗位**匹配**的专业认知；没有重大遗漏或理解偏差 | "准确说出了 Agent 记忆管理的基本概念和常见做法，但未做场景化深入" |
| **1** | 部分触及 | 部分触及问题，但遗漏了**主要考点**；或回答停留在表层，未能展示岗位所需的专业深度；或出现概念性偏差 | "知道 Agent 需要记忆，但把记忆管理和上下文窗口混为一谈，对核心概念理解有偏差" |
| **0** | 完全偏离 | 完全偏离问题核心，答非所问；或回答内容与岗位要求毫无关联；或出现严重的事实性/概念性错误 | "问 Agent 记忆管理，回答了数据库选型，完全不在同一个话题域" |

### 评分关键问题

- 候选人是否理解了面试官在问什么？（读懂题目）
- 回答是否覆盖了该问题预设的考点？（命中考点）
- 展示出的理解深度是否匹配岗位级别？（深度匹配）
- 是否有超出岗位预期的洞见？（加分项，非必须）

## 维度 2：结构清晰度（Structure Clarity）— 权重 25%

> **衡量的是**：回答的**逻辑组织**是否清晰、自洽、易于跟随。

### 评分锚定

| 分数 | 标签 | 标准 | 典型表现 |
|------|------|------|---------|
| **3** | 结构精良 | 结构精良，不仅自洽而且**高效**——用最少的表达传递最多的信息；听众可**轻松复述**其核心论点和论证链条 | "总论点→三个分论点→每个分论点有案例支撑→回扣总论点。听完后能清晰复述他的三个论点是什么" |
| **2** | 结构清晰 | 有明显的组织逻辑（总-分-总、STAR、金字塔）；观点之间有合理的递进或并列关系；**没有重大逻辑断裂** | "用了 STAR 框架，但 Situation 和 Task 之间有点跳跃，整体还能跟随" |
| **1** | 结构松散 | 有基本观点但缺少组织框架；前后逻辑**偶有断裂**；需要面试官追问才能理清思路 | "说了很多观点，但互相之间没有明显的逻辑关系，像散落的珠子，需要追问才能串起来" |
| **0** | 结构混乱 | 回答**混乱无逻辑线索**；观点散乱跳跃，无法跟随其思路；或自我矛盾、前后推翻 | "说了一半跳到另一个话题，又跳回来，听不出他在论证什么，也没有结论" |

### 评分关键问题

- 听到最后，我能用一句话复述他的核心观点吗？（可复述性）
- 观点之间的逻辑关系是明确的（并列/递进/因果）还是散乱的？（逻辑关系）
- 有没有一个清晰的组织框架（STAR/金字塔/总-分-总）？（框架意识）
- 因果链条完整吗——"因为→所以→导致→因此"链路完整？（因果完整性）

### 常见框架参考

| 框架 | 适用场景 |
|------|---------|
| STAR（Situation-Task-Action-Result） | 行为面试题、项目经验题 |
| 金字塔原理（结论先行→分论点→论据） | 观点阐述题、自我介绍 |
| 总-分-总 | 开放性论述题 |
| 问题→分析→方案→效果→复盘 | 案例分析题 |
| 概念→原理→应用→边界 | 专业知识题 |

## 维度 3：证据支撑度（Evidence Support）— 权重 25%

> **衡量的是**：回答是否有**具体案例、数据或经验**支撑，而非空洞的理论或套话。

### 评分锚定

| 分数 | 标签 | 标准 | 典型表现 |
|------|------|------|---------|
| **3** | 证据充分 | 案例详实且多样，能从多个角度佐证观点；数据引用精确（含基线、衡量方式、归因）；能清晰区分**"自己主导的"和"参与的"** | "'我主导了工具调用模块的优化，优化前成功率78%（基线：500次人工抽检），上线后提升至94%（同口径），净提升16pp。归因分析显示参数校验改造贡献了60%的效果。我也参与了推荐算法的选型，但主要负责需求侧。'" |
| **2** | 有据可查 | 有具体案例或数据支撑核心观点；案例可以追溯到**可验证的细节**（项目背景、具体行动、可量化结果）；但数据可能缺少基线或归因 | "'我们做了一个 Agent 平台，我负责设计了工具调用接口，上线后成功率提升到了94%。'"（有数据但缺基线和归因） |
| **1** | 证据模糊 | 有尝试举例，但案例过于模糊（"做过一个项目"而无法具体描述）；或数据引用不准确/不完整；或只有孤例 | "'我之前做过类似的项目，效果还不错。'"（完全无法判断项目规模、个人角色和实际成果） |
| **0** | 无证据 | 纯观点/纯理论，无任何具体例证；或引用的例证与论点无关；或数据明显编造 | "'我认为好的 AI 产品应该以用户为中心，重视体验，持续迭代。'"（正确但空洞，没有任何例证） |

### 评分关键问题

- 候选人是在"说话"还是在"证明"？（观点 vs 证据）
- 案例能否追溯到具体细节（项目名/时间/规模/角色）？（可验证性）
- 数据是否包含基线、衡量方式、归因？（数据完整性）
- 候选人能否区分"我主导的"和"我参与的"？（角色清晰度）

### 证据质量阶梯

```
无证据 ──→ 模糊案例 ──→ 具体案例 ──→ 具体案例+数据 ──→ 具体案例+完整数据+归因+角色清晰
  (0)        (1)          (2)            (2-3)                    (3)
```

## 维度 4：表达可信度（Expression Credibility）— 权重 15%

> **衡量的是**：回答的**真诚度和专业自信**——是在展示真实能力还是在背诵/包装。

### 评分锚定

| 分数 | 标签 | 标准 | 典型表现 |
|------|------|------|---------|
| **3** | 高度可信 | 表达极具说服力，让人感受到真实的**深度参与和思考**；对边界问题能给出诚实但有深度的回答（而非回避或硬撑）；展现出**清晰的自我认知**（知道自己的长处和短板） | "'这个问题我确实没有直接经验，但根据我对 Agent 通信协议的理解，我推断……不过我的推断需要在 XX 场景下验证才能确认'" |
| **2** | 自然真诚 | 表达自然、真诚；无明显背诵或模板化痕迹；能坦然承认**不了解**的部分并给出思考路径；回答体现出真实的项目参与感（细节感） | 语言自然，有细节但不过度，不确定时说"这部分我了解有限" |
| **1** | 偶有包装感 | 偶有模板化表达（但能回应具体问题）；部分内容有**包装感**——过度使用大词而无实质内容；或感觉在背诵准备好的答案 | "我深度赋能了业务增长，通过精细化运营和系统化思维驱动了核心指标提升"——全是术语，没有实质内容 |
| **0** | 明显背诵/矛盾 | 明显背诵模板；回答与问题不完全对应（疑似套用准备好的答案）；或出现**前后矛盾、自我推翻**；或编造痕迹明显 | 自我介绍和 Q3 的项目经验明显矛盾（时间线对不上、角色描述不一致） |

### 评分关键问题

- 听起来像是在聊天还是在念稿？（自然度）
- 当被追问到不熟悉的领域时，是坦诚还是回避/硬撑？（诚实度）
- 对个人贡献的描述是否可信——有细节感还是模糊包装？（真实性）
- 不同题目之间对同一经历的描述是否一致？（一致性）
- 是否有"我知道我不知道什么"的自我认知？（元认知）

## 评分校准规则

### 校准流程（标注前必做）

1. **锚定会议**：3 位标注者共同阅读 2 份标杆回答（一份强、一份弱），各自打分后讨论拉齐
2. **岗位级别校准**：确认本批次标注的岗位级别参照系（初级/中级/高级/专家）
3. **边界案例对齐**：对"2 分还是 3 分"的分歧点做明确约定（如"数据有基线但无归因 → 2 分"）

### 评分一致性要求

| 指标 | 目标 | 说明 |
|------|------|------|
| 单人重测信度 | ≥ 0.80 | 同一标注者间隔 7 天对同一份回答打分，Pearson r ≥ 0.80 |
| 评分者间信度（IRR） | ICC ≥ 0.70 | 3 位标注者对同一份回答的组内相关系数 |
| 分数偏差容忍度 | ≤ 1 分 | 任一维度上，任意两位标注者分差超过 1 分需讨论拉齐 |

### 拉齐规则

- 任意维度两位标注者分差 ≥ 2 → **强制讨论**，必须达成一致或由第三位 senior 标注者仲裁
- 三人分数全不一致（三个不同分数）→ 表决取中位数，若方差仍大则全组讨论
